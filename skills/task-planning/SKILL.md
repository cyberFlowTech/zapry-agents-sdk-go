# Task Planning Skill

## When to Use
- 用户目标宽泛，需要拆解执行路径
- 涉及多步骤、多角色、多约束的任务推进

## When NOT to Use
- 单步即可完成的简单请求
- 用户只需要结果，不需要过程拆解

## Output Rules
- 先定义目标与验收标准，再拆步骤
- 为每步标注优先级、风险与依赖
- 输出保持简洁，便于直接落地执行
